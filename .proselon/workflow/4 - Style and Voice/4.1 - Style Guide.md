# Style Guide Template

Use this template to create a project style guide once the series plot exists. The style guide formalizes prose decisions relative to genre conventions documented in `Plot/Specs.md`.

Every prose decision should be either following a genre convention or deliberately subverting one for a stated reason tied to plot, theme, or character. No prose choice should be accidental.

## Interview Goals

Before filling the template, gather enough information to answer:

- What narrative register fits the story's characters and themes?
- Where does the project follow genre prose conventions from `Plot/Specs.md`?
- Where does the project depart from genre conventions, and what does each departure serve?
- What POV and tense choices best serve the story?
- What vocabulary and terminology rules does the project need?
- What prose patterns should the project avoid?

Do not draft example prose during the interview unless the author explicitly asks.

## Context to Load

Read these files before beginning the interview:

1. `Plot/Specs.md` — especially the Prose Style Conventions section for genre norms
2. `Plot/Series Plot.md` — for series-level tone and themes
3. `Plot/Themes and Conflict.md` — for thematic material the style should reinforce
4. Any character profiles in `Worldbuilding/Characters/` — for voice, register, and vocabulary considerations (if any have been created)
5. Any existing `Style/Style Guide.md` — if revising rather than creating

## Completion Criteria

A usable style guide:

- References the genre convention from `Plot/Specs.md` for every prose dimension.
- States a clear project decision (follow or subvert) for each dimension.
- Provides a plot, theme, or character justification for every subversion.
- Gives rules specific enough for a drafting agent to follow without ambiguity.
- Includes a Red Flag List derived from genre conventions and voice boundaries.
- Consolidates all departures in the Intentional Subversions Summary table.
- Does not contradict the Voice Reference (`Style/Voice Reference.md`) if it already exists.
