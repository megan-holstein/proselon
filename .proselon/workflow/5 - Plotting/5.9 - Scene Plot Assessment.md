# Scene Plot Assessment

Evaluate all scene plots in a chapter as a set. Run this rubric only when the chapter plot is in place.

## Prerequisites

Read these files before assessing:

1. Every scene plot file in the plot chapter directory
2. The matching chapter plot (`Plot/Book N/Chapters/Chapter N/Chapter N Plot.md`) — the psychological question and planned emotional shift the scenes must deliver
3. `Plot/Book N/Story State.md` — the authoritative cross-scene state record; verify Start States and Information Log provenance against it
4. The previous chapter's final scene plot (and its manuscript file, if drafted) — for line-level continuity into S1 only *(none for the book's first chapter)*
5. Project character profiles for all characters present across the scenes *(those that exist)*
6. Project worldbuilding, setting, continuity, and genre-system references *(those that exist)*
7. `Plot/Specs.md` — word count targets

Assess against what exists at this point in the workflow. A missing file marked conditional above is not a weakness; skip the criteria that depend on it.

## Procedure

Assess each scene individually, then assess the set as a whole.

### Per-Scene Checklist

For each scene plot, check:

- **Scene purpose**: The scene delivers a specific narrative function (revelation, relationship shift, escalation, decision). If you cannot name its purpose in one sentence, the scene may not earn its place.
- **Try-fail cycle**: The character attempts something. It does not go as planned. The outcome creates a new situation — not a reset to the starting state.
- **Continuity**: Time of day, injuries, objects, clothing, weather, and emotional state are consistent with the ledger (`Plot/Book N/Story State.md`) first, and with the previous scene second. Start States and Information Log provenance claims must match the ledger — do not reconstruct state by walking prior scene plots; that's what the ledger is for.
- **Value change**: The scene shifts at least one value for the POV character (e.g., safety to danger, trust to suspicion, hope to despair). A scene where nothing changes is a candidate for cutting.
- **Internal conflict**: The scene identifies a specific psychological tension — competing desires, beliefs, or instincts — that the POV character is caught between. The internal conflict connects to the chapter plot's psychological question and is pressured by the scene's external obstacle. A scene where the character only reacts to external events without internal tension is a candidate for strengthening.
- **Word count target**: Uses the scene word-count range specified in `Plot/Specs.md` as a planning guide. Individual scenes may go under or over when the material earns it; flag deviations only when they create pacing, proportion, or chapter-length problems.

### Set-Level Checklist

After assessing each scene individually, evaluate the full set:

- **Chapter emotional arc**: The scenes together deliver the emotional shift planned for the chapter. The arc has shape — it builds, turns, and lands.
- **Internal conflict progression**: Across the chapter's scenes, the internal conflict deepens, complicates, or transforms — it doesn't merely repeat the same tension at the same intensity. By the chapter's final scene, the psychological pressure should be measurably different from where it started.
- **Handoff continuity**: Each scene picks up where the previous one ends. No gaps in time, location, or emotional state that aren't accounted for.
- **No duplicate purposes**: Each scene serves a distinct function. If two scenes do the same narrative work, one is a candidate for cutting or merging.
- **Total word count**: The sum of scene word counts keeps the chapter's likely scale visible against the chapter target from `Plot/Specs.md`, but the target is advisory. Flag over/under totals only when they create pacing, proportion, or book-length problems.
- **Genre or story-system consistency**: Any magic, technology, politics, law, profession, institution, relationship rule, mystery logic, social system, or other governing rules across the scenes follow established project rules.
- **Worldbuilding integration**: Worldbuilding arrives through character experience, not exposition blocks. If a scene exists primarily to deliver worldbuilding, flag it.

## Output Format

```
## Scene Plot Assessment: [Chapter Name]

### Per-Scene

**S1 - [Scene Name]**
Purpose: [one sentence]
Verdict: Approved / Needs revision
Issues: [if any]

**S2 - [Scene Name]**
...

### Set Assessment

**Strengths** (ranked)
1. ...

**Weaknesses** (ranked by severity)
1. ...

**Required Revisions**
- [Only include if verdict is Needs revision. Each item should be specific enough to edit a named scene plot.]

**Verdict**: Approved / Needs revision
```

Concise, ranked, actionable. If the verdict is Needs revision, weaknesses must include specific enough detail to act on.

Use Needs revision only for problems drafting would inherit — a scene without a purpose or value change, a continuity claim that contradicts the ledger, a broken chapter arc, or duplicate scenes. Do not require prose-level detail the draft will supply.
