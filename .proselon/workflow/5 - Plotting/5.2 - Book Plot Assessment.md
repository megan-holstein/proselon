# Book Plot Assessment

Evaluate structure within a single book — the next book to write or revise. Run this rubric only when the series plot is in place.

This rubric runs as a **panel of three independent Assessors** (see AGENTS.md › Panel Assessment for Planning Gates): if you are reading this as an Assessor, you are one of three, unaware of the others — run the rubric alone and return your own findings and verdict.

## Prerequisites

Read these files before assessing:

1. `Plot/Book N/Book N Plot.md` — detailed chapter-by-chapter outline for the target book
2. `Plot/Book N/Story State.md` — the book's ledger; the gate is "both exist," so assess it too
3. `Plot/Series Plot.md` — for series-level context (where this book sits in the arc)
4. `Plot/Specs.md` — word count targets and pacing guidelines
5. Project character profiles for all characters appearing in this book *(those that exist)*
6. `Plot/Themes and Conflict.md` — for thematic thread tracking
7. `Style/Style Guide.md` — the project's POV rules, for the Multi-POV criterion
8. For Book 2 onward: the previous book's final `Plot/Book N-1/Story State.md` — for the carry-forward criterion

Assess against what exists at this point in the workflow. A missing file marked conditional above is not a weakness; skip the criteria that depend on it.

## Procedure

Work through every criterion below. The checklist is your internal rubric — only findings appear in the output.

### Checklist

- **Inciting incident placement**: Occurs within the first 15% of word count. The protagonist's world is disrupted clearly and irreversibly.
- **Progressive complications**: The middle 50% contains escalating obstacles. Each complication raises stakes or narrows options. No sagging middle — if a chapter could be removed without consequence, the middle sags.
- **Crisis / turning point**: A clear moment (usually around 75%) where the protagonist faces a forced choice — two paths, both costly.
- **Climax payoff**: The climax resolves the book's central question. Setups from earlier chapters pay off here.
- **Resolution**: Brief. Establishes the new normal. Plants forward momentum for the next book, unless this is the final book and should close the series.
- **Subplot tracking**: Every subplot introduced is either resolved within the book or explicitly carried forward. No orphaned subplots.
- **Internal conflict escalation**: The protagonist's internal conflict — the competing desires or beliefs the theme creates — escalates progressively alongside external stakes. Psychological pressure deepens; the character is not merely reacting to events but increasingly caught between irreconcilable demands.
- **Theme-to-arc connection**: Each active theme in `Plot/Themes and Conflict.md` manifests through at least one character's internal conflict, not just through plot events, dialogue, or exposition. The thematic arc map in the book plot identifies these connections.
- **Freshness**: The book's major beats are not genre boilerplate. If the inciting incident, midpoint, and climax could be transplanted unchanged from any comp title in `Plot/Specs.md`, the plot is executing the genre's default path — at least one major beat should resolve differently than a genre-savvy reader would predict.
- **Pacing by quarter**: First 25% establishes and hooks. Middle 50% complicates and escalates. Final 25% accelerates to climax and resolves.
- **Chapter-to-chapter momentum, in varied ways**: Each chapter ends with something that pulls the reader forward — a question, decision, revelation, threat, or unresolved emotional weight. Deliberate quiet or aftermath endings are legitimate where the genre norms in `Plot/Specs.md` support them: a reflective close that leaves weight unresolved still pulls. What fails is an ending at genuine rest — *and* an unbroken run of the same hook class. Thirty consecutive cliffhangers are themselves a formula readers feel; if a long run of chapters ends on the same kind of pull, flag the monotony as a weakness.
- **Multi-POV management**: If secondary POVs are used, each serves a narrative purpose that the primary POV cannot. POV transitions follow the project's POV rules, and no character knows things they could not know.
- **Terminology and perspective**: Project-specific terms, titles, factions, and historical events are named according to the thinking or speaking character's perspective.
- **Ledger initialized**: `Plot/Book N/Story State.md` exists and meets the Completion Criteria in the Story State template (5.3) — seeded with the book's starting conditions, no prose, no duplicated canon.
- **Cross-book carry-forward** *(Book 2 onward)*: Every open thread and end-state condition in the previous book's final ledger is either carried into this book's ledger (with original provenance) or deliberately closed — nothing silently dropped between books.

## Output Format

```
## Book Plot Assessment: [Book Title]

**Strengths** (ranked)
1. ...

**Weaknesses** (ranked by severity)
1. ...

**Required Revisions**
- [Only include if verdict is Needs revision. Each item should be specific enough to edit the book plot.]

**Verdict**: Approved / Needs revision
```

Concise, ranked, actionable. If the verdict is Needs revision, weaknesses must include specific enough detail to act on.

Use Needs revision only for structural issues that chapter-level work would inherit — a missing or misplaced turning point, a sagging middle, orphaned subplots, internal conflict that never escalates, an uninitialized ledger, or a plot whose major beats a genre-savvy reader would fully predict (a Freshness failure — predictability at this level is unfixable downstream and blocks like any structural defect). Do not require chapter-level detail the Chapter Map (5.4) will supply.

When the failing criterion is Freshness, the Required Revision should direct **regeneration** — fresh structural alternatives to compare against — rather than in-place edits: patching one beat of a genre-default plot yields the default plot with a twist.
