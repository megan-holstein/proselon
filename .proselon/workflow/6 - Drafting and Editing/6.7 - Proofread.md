# Proofread

Tier: mechanical — pattern-matching against explicit criteria; a fast, inexpensive model is adequate.

This is the final polish. The mechanics are clean, the prose is refined, and the facts have been verified (6.5). You are checking manuscript-file formatting and doing one last readthrough for snags.

## Before You Start

Load only:

- This pass doc
- The current scene file
- The project style guide (`Style/Style Guide.md`) — its formatting rules

## Formatting

Per the project style guide:

- **No headers** at the start of scene files. The prose begins immediately.
- **No planning notes** in manuscript files. Scene briefs, outlines, editorial notes, and stubs belong outside the exportable manuscript tree.
- **Single line breaks** between paragraphs (one blank line, not two).

## Final Readthrough

Read the scene one last time from beginning to end. Does it flow? Does anything snag? If you have to re-read a sentence to parse it, it needs attention — even if it's technically correct.

## How to Make Changes

Fix formatting violations silently — they are mechanical. Do **not** rewrite snagging sentences yourself: smoothing a sentence is voice work, above this pass's tier. List each snag in the final response (quote the sentence) so the orchestrating agent can route it to a craft-tier fix. If nothing snags, say so.

Preserve exportable manuscript content only; do not embed scene plots or planning notes in the manuscript file.
